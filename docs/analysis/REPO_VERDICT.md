# Обзор репозиториев и вердикт

Дата обзора: 4 февраля 2026

Ниже — понятное описание, за что отвечает каждая часть проектов, и практичный вердикт по применимости.

---

## 1. DeepAnalyze

### Что это
Agentic‑LLM для автономной аналитики данных (полный цикл: подготовка данных → анализ → моделирование → визуализация → отчеты). Можно запускать локально; есть API, CLI, WebUI и Jupyter‑интерфейс. Проект ориентирован на исследование и самостоятельный хостинг.

### Что за что отвечает (структура)
- `API/` — сервер API в стиле OpenAI (chat/completions + файлы), управление рабочими папками `workspace/` и потоками диалога.
- `demo/` — интерфейсы и демо:
  - `demo/chat` — WebUI (frontend + backend) для работы через браузер.
  - `demo/cli` — CLI-интерфейс для терминала.
  - `demo/jupyter` — интеграция с Jupyter‑Lab.
- `deepanalyze/` — внутренняя часть модели/обучения/утилит, включая подпроекты `ms-swift` и `SkyRL`.
- `playground/` — бенчмарки и тестовые наборы (DSBench, DS‑1000, TableQA и т.д.).
- `example/` — примеры анализа и демонстрационных кейсов.
- `docs/` — документация, FAQ, гайд по API‑ключам.
- `docker/` — контейнеризация и помощь с окружением.
- `scripts/`, `run.py`, `quantize.py` — вспомогательные утилиты и запуск/квантизация.

### Стек/запуск (вкратце)
- Python + vLLM для поднятия модели
- Node.js для WebUI
- Требуются GPU ресурсы (в README есть таблица рекомендуемых параметров памяти)

### Вердикт
Сильная исследовательская база и набор интерфейсов. Подходит для продвинутых пользователей, которым нужен автономный DS‑агент локально. Основные риски: тяжелые требования к GPU и инженерная сложность развертывания.

## 2. ValueCell

### Что это
Мультиагентная платформа для финансового анализа и трейдинга: исследования, новости, стратегии, интеграции с биржами. Ориентировано на десктоп‑приложение с локальным хранением данных и подключением к разным LLM‑провайдерам.

### Что за что отвечает (структура)
- `python/valuecell/` — основное ядро на Python:
  - `core/` — оркестрация, планирование, маршрутизация ответов.
  - `agents/` — агенты (исследования, новости, стратегии и т.д.).
  - `adapters/` — интеграции с источниками данных/биржами.
  - `server/` — backend API.
  - `config/`, `utils/` — конфиги и утилиты.
- `frontend/` — интерфейс и десктоп‑оболочка:
  - `src/` — React UI.
  - `src-tauri/` — Tauri‑обертка для десктопа.
- `docs/` — архитектура, конфигурация, гайды для разработчиков.
- `docker/` — контейнерные сценарии.
- `assets/` — скриншоты/диаграммы.

### Стек/запуск (вкратце)
- Backend: Python
- Frontend: React + Vite + Tauri
- Запуск всего стека: `start.sh` / `start.ps1`
- Требуются ключи LLM‑провайдеров и (опционально) биржевые API‑ключи

### Вердикт
Полноценный продукт с серьезной архитектурой и фокусом на финансы. Полезно для исследований и полуавтоматизации, но любые live‑торги — зона высокого риска; нужно четко настроить ограничения, тестовые окружения и безопасность ключей.

## 3. Open‑Interface

### Что это
Автопилот компьютера на базе LLM: отправляет скриншоты в модель, получает инструкции, выполняет их через клавиатуру/мышь. Работает на macOS/Linux/Windows (есть готовые сборки). Это инструмент именно для управления интерфейсом, не для написания кода.

### Что за что отвечает (структура)
- `app/` — основное приложение на Python:
  - `ui.py` — GUI.
  - `core.py` — главный цикл: скриншот → LLM → действия.
  - `llm.py` — соединение с LLM‑провайдерами.
  - `interpreter.py` — интерпретация инструкций модели.
  - `utils/`, `resources/`, `models/` — вспомогательные модули.
- `assets/` — медиа/скриншоты.
- `tests/` — тесты.

### Стек/запуск (вкратце)
- Python 3.12, зависимости из `requirements.txt`
- Требуются права на управление экраном/клавиатурой (Accessibility + Screen Recording на macOS)
- Поддержка OpenAI/Gemini или совместимых API

### Вердикт
Подходит для автоматизации пользовательских сценариев (клик‑путь, заполнение форм, навигация). Вещь экспериментальная: иногда ошибается в кликах и пространственной логике, требует присмотра и повышенных разрешений ОС.

## 4. Auto‑Claude

### Что это
Автономный фреймворк для разработки ПО с агентами: планирование задач, реализация, тесты/QA, работа через изолированные git‑worktree. Это больше про управление разработкой, чем про «клик по экрану».

### Что за что отвечает (структура)
- `apps/backend/` — сердце системы (Python): агенты, планирование, QA‑циклы, memory‑слой, интеграции (Linear, GitHub/GitLab).
- `apps/frontend/` — десктоп‑приложение (Electron).
- `guides/` — гайды (CLI, Linux builds и т.п.).
- `tests/` — тесты.
- `scripts/` — сборка/утилиты.
- `.design-system/` — дизайн‑система UI.

### Стек/запуск (вкратце)
- Backend: Python 3.10+
- Frontend: Electron
- Требуется Claude Pro/Max + `@anthropic-ai/claude-code`

### Вердикт
Зрелый инструмент для командной разработки и автономных build‑циклов. Сильные стороны — изоляция в worktrees и QA‑контур. Минус — высокий порог входа, зависимость от Claude‑подписки и относительно «тяжелый» стек.

---

## 5. Taskhomie (computer-agent)

### Что это
Локальный агент, который управляет компьютером: скриншоты, мышь/клавиатура, терминал, браузер. Два режима: полный контроль экрана и фоновая веб‑автоматизация (CDP).

### Что за что отвечает (структура)
- `src/` — интерфейс (React + Tailwind), окна, состояния.
- `src-tauri/` — backend на Rust (Tauri):
  - `agent.rs` — основной агент (общение с API, управление режимами).
  - `computer.rs` — управление мышью/клавиатурой.
  - `browser.rs` — управление браузером через DevTools.
  - `bash.rs` — выполнение команд.
  - `voice.rs` — голосовые функции.
  - `storage.rs` — история/состояние.

### Стек/запуск (вкратце)
- Frontend: React + Vite
- Backend: Rust + Tauri
- Нужен Anthropic API key

### Вердикт
Хорош для исследовательской автоматизации (десктоп + браузер). Как и все компьютер‑агенты — рискованный для «без присмотра»: требует прав ОС, может ошибаться, нужен стоп‑контроль.

# Итоговый вывод

- **Для автономной аналитики данных** — DeepAnalyze (при наличии GPU и опыта развертывания).
- **Для фин. агентов и трейдинг‑экспериментов** — ValueCell (осторожно с реальной торговлей).
- **Для автопилота интерфейса** — Open‑Interface или Taskhomie (разные стеки, похожие риски).
- **Для автономной разработки ПО** — Auto‑Claude (самый «продуктовый» и взрослый, но тяжелый и платный).
